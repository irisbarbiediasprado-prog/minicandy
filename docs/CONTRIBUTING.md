# Fluxo de desenvolvimento

## Um commit = uma feature

Toda feature deve conter:

- implementação
- validação
- documentação

## Regra principal

Nunca criar um commit antes da implementação existir.

## Fluxo

Sugestão

↓

One-liner

↓

Validação

↓

Commit

↓

Push

↓

Próxima feature

## Filosofia

A IA carrega a complexidade.

A Íris carrega a criatividade.

Toda tarefa repetitiva deve virar um comando do mc.
